# Техническое задание: LNN-based Low-Light Video Enhancement

**Версия:** 0.1  
**Дата:** 2026-06-05  
**Статус:** Draft  

---

## 1. Цель проекта

Исследовать применимость Liquid Neural Networks (LNN) как компонента для моделирования временно́й динамики освещения в задаче улучшения видео при слабом освещении (Low-Light Video Enhancement, LLVE).

**Основная гипотеза:**  
Непрерывная ODE-динамика LNN лучше моделирует физику изменения illumination map во времени, чем дискретные рекуррентные сети (LSTM/GRU), что приводит к улучшению метрик качества и временно́й консистентности.

**Вторичная гипотеза:**  
Обученный LNN-модуль может быть переиспользован как компактная суррогатная модель динамики освещения для real-time рендеринга (игры, XR-приложения).

---

## 2. Контекст и мотивация

### 2.1 Проблема
Существующие методы LLVE (RetinexFormer, NAFNet и др.) обрабатывают каждый кадр независимо или используют дискретные рекуррентные сети для временно́й связности. Изменение освещения — физически непрерывный процесс, описываемый ODE. LNN (и их closed-form вариант CfC) решают ODE явно, что теоретически соответствует природе задачи.

### 2.2 Новизна
- Применение LNN/CfC к LLVE не описано в литературе (по состоянию на июнь 2026)
- Интеграция LNN в Retinex-декомпозицию (между illumination estimator и damage restorer) — новая архитектурная идея
- Сравнение LNN vs LSTM/GRU как темпоральных компонентов в одинаковом пайплайне — измеримый вклад

### 2.3 Смежные работы
- **RetinexFormer** (ICCV 2023, NTIRE 2024 Runner-up, NTIRE 2025/2026 Winner) — базовая архитектура
- **Vid-ODE** (ODE-ConvGRU для видеогенерации) — структурный аналог
- **HSI-CfC** (CNN + CfC для гиперспектральных изображений) — паттерн интеграции
- **TempRetinex** (unsupervised Retinex для видео) — ближайший конкурент по задаче
- **TransGI / Neural Light Transport** — контекст для суррогатного применения

## 3. Архитектура

### 3.1 Основная архитектура: Retinex-LNN

```
Входное видео (кадры t-N ... t)
         │
         ▼
┌─────────────────────┐
│  CNN Encoder        │  ← на основе RetinexFormer Illumination Estimator
│  (RetinexFormer)    │    выдаёт: illumination_map_t, reflectance_map_t
└─────────────────────┘
         │
   illumination_map_t ──────────────────────────┐
         │                                       │
         ▼                                       ▼
┌─────────────────────┐              ┌─────────────────────┐
│  LNN (CfC)          │              │  Reflectance        │
│  Temporal Module    │              │  (передаётся        │
│                     │              │   напрямую)         │
│  Моделирует ODE     │              └─────────────────────┘
│  динамики света     │                        │
└─────────────────────┘                        │
         │                                     │
   illum_enhanced_t                            │
         │                                     │
         └──────────────┬──────────────────────┘
                        ▼
             ┌─────────────────────┐
             │  CNN Decoder        │  ← RetinexFormer Damage Restorer
             │  (RetinexFormer)    │
             └─────────────────────┘
                        │
                        ▼
               Enhanced frame_t
```


### 3.2 Варианты LNN-модуля

| Вариант | Описание | Когда использовать |
|---|---|---|
| **CfC** (Closed-form Continuous-time) | Аналитическое решение LTC, быстрее | Основной вариант |
| **LTC** (Liquid Time-Constant) | Оригинальный, ODE-solver, медленнее | Ablation study |

### 3.3 Baseline для сравнения

Та же архитектура с заменой LNN-модуля:
- **LSTM** — стандартный baseline
- **GRU** — лёгкий baseline  
- **Без темпорального модуля** — нижняя граница (per-frame RetinexFormer)

## 4. Фазы реализации

### Фаза 1 — Baseline (3–4 недели)
**Цель:** воспроизвести RetinexFormer на видео как per-frame baseline

- [ ] Настроить окружение (conda, pytorch, зависимости RetinexFormer)
- [ ] Клонировать и запустить RetinexFormer на LOL dataset
- [ ] Адаптировать под видеовход (dataloader для BVI-RLV)
- [ ] Получить baseline метрики: PSNR, SSIM, LPIPS на BVI-RLV

**Выход:** работающий RetinexFormer на видео, baseline числа

### Фаза 2 — LSTM/GRU интеграция (2–3 недели)
**Цель:** внедрить дискретный темпоральный модуль, установить сильный baseline

- [ ] Встроить LSTM между illumination estimator и damage restorer
- [ ] Обучить на BVI-RLV
- [ ] Метрики + temporal consistency metric (warping error)
- [ ] Повторить для GRU

**Выход:** CNN+LSTM и CNN+GRU числа — прямые конкуренты для LNN

### Фаза 3 — LNN интеграция (3–4 недели)
**Цель:** заменить LSTM на CfC, сравнить

- [ ] Установить ncps (neural circuit policies) — официальная библиотека LNN
- [ ] Реализовать CfC-модуль с тем же интерфейсом, что LSTM
- [ ] Обучить, получить метрики
- [ ] Ablation: LTC vs CfC

**Выход:** основной результат исследования

### Фаза 4 — Анализ и визуализация (1–2 недели)
- [ ] Qualitative comparison (видеопримеры)
- [ ] Ablation study (размер окна, число слоёв LNN)
- [ ] Визуализация illumination map dynamics
- [ ] Анализ скорости инференса

## 5. Датасеты и бенчмарки

| Датасет | Тип | Размер | Назначение |
|---|---|---|---|
| **BVI-RLV** | Видео, paired | 31 800 кадров, 1080p | Основной train/test |
| **LOL** | Изображения, paired | 500 пар | Совместимость с RetinexFormer |
| **VE-LOL** | Изображения, paired | 13 440 изображений | Дополнительный тест |

### Метрики
- **PSNR** — пиковое соотношение сигнал/шум
- **SSIM** — структурное сходство
- **LPIPS** — перцептивное сходство
- **Warping Error** — временна́я консистентность между кадрами
- **Inference time** (мс/кадр на стандартном GPU)

## 8. Открытые вопросы (требуют решения в Фазе 1)

1. **Размер временно́го окна** — сколько кадров подавать в LNN? (3, 5, 7)
2. **Пространственное сжатие** — как свернуть spatial illumination map в вектор для LNN: global avg pool, spatial attention, patch tokens?
3. **∆t для ODE** — использовать реальный временной интервал между кадрами (1/fps) или нормализованный?
4. **Causal vs non-causal** — однонаправленный LNN (real-time ready) или bidirectional (лучше качество)?

## 9. Ожидаемые результаты

### Минимальный результат (успех проекта)
- LNN не хуже LSTM по PSNR/SSIM при меньшем числе параметров
- Статья/отчёт с ablation study

### Целевой результат
- LNN превосходит LSTM на 0.3+ дБ PSNR
- Улучшение warping error (temporal consistency)
- Анализ interpretability: визуализация ODE-траекторий illumination

### Бонусный результат
- Дообученный LNN-модуль как суррогатная модель для динамики освещения
- Демонстрация применимости в real-time контексте (< 15 мс/кадр)
